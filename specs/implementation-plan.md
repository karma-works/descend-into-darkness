## Phase 1 — Audio Engine

**Goal:** Spatial audio working in isolation. Can place entities in the audio field, emit echolocation pings, hear the ambient cave.

**Files:**
- `src/SpatialAudio.ts`
  - `AudioContext` with `MasterGain`, reverb `ConvolverNode`
  - `EntityAudioSource`: `PannerNode` + `GainNode` per entity
  - `updateEntity(id, x, y, entityType)` — repositions panner in real-time
  - `removeEntity(id)` — disconnects and cleans up
  - `emitEcholocationPing(entities[])` — fires spatial click per entity, 100ms total
  - `startAmbient(depth)` — cave drone, pitch/dissonance varies with depth
  - `playSignature(entityType)` — one-shot audio for contact/collect events
- `src/ThreatBus.ts`
  - `emit(event: ThreatEvent)` — routes to CRITICAL / TACTICAL / AMBIENT handler
  - CRITICAL: fires immediately, suppresses TACTICAL
  - TACTICAL: debounced queue, 400ms minimum between items, flushes to `MaraEngine`
  - AMBIENT: continuous, never interrupted

**Acceptance:** Headphone test — two entities placed left and right, player pings, hears distinct left/right spatial response. Ambient starts and varies in character when depth parameter changes.

## Phase 2 — Game World

**Goal:** Level structure, physics, and game loop skeleton. No rendering — pure data and console logs verify state.

**Files:**
- `src/types.ts` — `GameState`, `EntityType`, `ThreatLevel`, `LevelData`, `PhysicsBody`
- `src/constants.ts` — `LEVEL_WIDTH`, `TILE_SIZE`, `GRAVITY`, `REX_SPEED`, `DEBRIS_WARN_MS = 1800`, `ECHOLOCATION_COOLDOWN_MS = 2000`, `BOSS_LEVELS`
- `src/Physics.ts` — tile-based terrain grid, AABB collision, gravity, jump force
- `src/Level.ts` — procedural generator: terrain tiles, entity spawn points, merchant position, boss trigger, exit position, checkpoint positions
- `src/Input.ts` — keyboard handler (Q for echolocation, all controls from ADR-006)
- `src/Game.ts` (skeleton) — state machine, `update(dt)`, entity list, collision checks stubbed

**Acceptance:** `bun run dev`, open browser, game enters PLAYING state, Rex moves left/right and jumps on a generated terrain grid (verified via console log of position).

## Phase 3 — Mara Engine

**Goal:** Mara speaks. Arc-aware. Post-death narration works. Echolocation announces results.

**Files:**
- `src/MaraEngine.ts`
  - `speak(line: string, priority: ThreatLevel)` — queues or interrupts based on priority
  - `onDeath(cause: string, direction: string, missedSignal: string)` — immediate post-death narration
  - `onEcholocationResult(entities: SpatialResult[])` — translates spatial data to language
  - `onDepthChange(depth: number)` — selects arc-appropriate voice pool
  - `onProactiveThreat(type: string, direction: string, distance: string)` — called by ThreatBus TACTICAL
  - `onShopEnter(gems: number, items: ShopItem[])` — Mara's shop intro with opinion
  - `onSilence()` — suppresses Mara's voice (wraith mechanic)
  - Line pools: `LINES_SURVEY`, `LINES_DARK`, `LINES_DESCENT`, `LINES_BOSS` — written dialogue
- Speech synthesis: `window.speechSynthesis`, voice selection prefers a female voice, rate/pitch adjusted per arc state

**Acceptance:** Start game, die to debris, hear Mara's post-death narration with correct directional information. Press Q, hear echolocation description of surroundings. Advance to depth 5, notice Mara's tone shift.

## Phase 4 — Entities

**Goal:** All entity types implemented with full audio signatures and gameplay behavior.

**Files:**
- `src/entities/Rex.ts` — movement, jump (with double-jump upgrade), fire, echolocation emitter, HP, upgrades (shield, fire level, speed level)

- `src/entities/Crystal.ts` — static collectible, spatial chime, gem value
- `src/entities/Merchant.ts` — stationary NPC, triggers shop state on `E` key when near, Mara voices all shop interaction
- `src/entities/Boss.ts` — Phase 1 / Phase 2; breathing audio arc; debris request pattern; charge attack with 600ms telegraph; death silences breathing

**Acceptance:** Full run-through of Level 1 (debris only), Level 2 (crawler introduced), Level 5 (first boss). Headphone test at each stage verifies spatial audio encoding.

## Phase 5 — ARIA Integration

**Goal:** Screen reader users receive all discrete game events via live regions.

**Files:**
- `src/ARIALog.ts`
  - `logEvent(message: string)` — appends `<p>` to `#event-log` with 500ms debounce
  - `alertCritical(message: string)` — sets `#mara-alert` text (assertive, interrupts screen reader)
  - All event types from ADR-006 wired through `ARIALog`
  - Post-death narration mirrored to `#event-log` (text of what Mara spoke)

**Acceptance:** Run game with NVDA or VoiceOver. Checkpoint reached → screen reader announces it. Treasure collected → announced. Death → post-death narration read aloud via screen reader (independent of Web Speech API TTS). No queue flooding.

## Phase 6 — Polish and Difficulty Curve

**Goal:** Game is genuinely playable and teachable. Difficulty curve front-loads audio learning.

**Difficulty progression:**

| Levels | New mechanic introduced | Mara's teaching style |
|---|---|---|
| 1–2 | Debris only. Wide platforms. No enemies. | Very chatty, explains echolocation explicitly |
| 3–4 | Crawler introduced. | Narrates crawler position, rhythm, patrol pattern |
| 5 | First boss. | Pre-boss: "I've seen this before in the fossil record." |
| 6–8 | Bat swarms. | Warns about speed, teaches "move before contact range" |
| 9 | Shadow Wraith first appearance. | Goes quiet. Describes nothing. Player must figure it out. |
| 10 | Boss 2, increased debris rate. | Mara's arc shifts to "The Descent" voice |
| 11+ | Elite enemy variants, near-instant debris. | Mara mostly silent. Occasional dark humor only. |

**Polish items:**
- Mara silence mechanic fully tied to wraith proximity
- Boss breathing degradation properly encoded
- Echolocation cooldown visual indicator (HTML text only: "Ping ready")
- Depth record persistence via `localStorage`
- All Mara line pools reviewed for tone and arc consistency

## Key Tuning Constants (initial values, adjust in Phase 6)

| Constant | Value | Rationale |
|---|---|---|
| `DEBRIS_WARN_MS` | 1800 | Proven in DINO. Calibrated for audio reaction time. |
| `ECHOLOCATION_COOLDOWN_MS` | 2000 | Prevents ping abuse; forces spatial audio reliance |
| `REX_SPEED` | 4 tiles/s | Same as DINO base speed |
| `GRAVITY` | 0.55 | Identical to DINO |
| `MARA_TACTICAL_DEBOUNCE_MS` | 400 | Minimum gap between Mara voice lines |
| `AMBIENT_SUPPRESSION_RANGE` | 300 tiles | Wraith triggers silence at this distance |
| `BOSS_LEVELS` | 5, 10, 15, 20… | Same cadence as DINO |
| `CRAWLER_BPM` | 80 | Slow, learnable rhythm |
| `BAT_BPM` | 200 | Chaotic, fast, punishing |
